"<プラグイン名>.lrdevplugin"という名称のディレクトリを作成
作成したディレクトリ配下に"Info.lua"というファイルを作成.
このファイルがプラグインのManifestファイルとなる

ローカライゼーション
Lightroom SDKにはZStringを用いた多言語対応のフレームワークが備わっている.
Info.luaと同階層にTranslatedStrings_ja.txtというファイルを作成し、
その中に翻訳を記述していく.
もちろん, 日本語なら    TranslatedStrings_ja.txt,
          英語なら      TranslatedStrings_en.txt,
          フランス語ならTranslatedStrings_fr.txt
となる.
このTranslatedStrings_xx.txtファイルは, UTF-8エンコーディングで記述する必要がある.
ファイルのフォーマットは,
"$$$/hogehoge/fugafuga/foo=にほんご"
"$$$/hogehoge/piipoo/bar=にほんご2"
という用な感じ.
quoteで囲んで, $$$からはじまって, パスを指定. パスと"="で結ぶ形で翻訳を記述.
この「パス」でマッチされた語がLightroomのロケールに併せて翻訳される.

肝心のluaスクリプト内の翻訳される側の文字列は, 以下のように記述
LOC "$$$/hogehoge/fugafuga/foo=english"
LOC ("$$$/hogehoge/piipoo/bar=english2" )
LOCからはじまって, quoteで囲まれた$$$から始まる文字列.
この"="記号のあとの文字列は, 翻訳がない場合のデフォルト文字列.

自作のプラグインを選択した時にステータス項目の上に独自設定メニューを表示させるなら,
Info.luaに
    LrPluginInfoProvider = 'PluginInfoProvider.lua',
という行を追加し, PluginInfoProvider.luaの中で

local function sectionsForTopOfDialog( f, _ )
    return {
        -- Section for the top of the dialog.
        {
            title = LOC "$$$/GeoMoves/PluginManager/Title=Plugin Settings",
            f:row {
                spacing = f:control_spacing(),

                f:static_text {
                    title = LOC "$$$/GeoMoves/PluginManager/NeedSignin=Sign-in to Moves to use this plugin.",
                    fill_horizontal = 1,
                },

                f:push_button {
                    width = 150,
                    title = LOC "$$$/GeoMoves/PluginManager/ButtonTitle=Connect to Moves",
                    enabled = true,
                    action = function()
                        LrHttp.openUrlInBrowser( PluginInit.URL )
                    end,
                },
            },
        },

    }
end

return{

    sectionsForTopOfDialog = sectionsForTopOfDialog,

}
